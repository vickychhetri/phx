PHX Developer Guide

Welcome to the complete language and runtime reference for PHX. Build native, high-performance, and concurrent cloud-ready systems using a familiar PHP syntax compiled directly to Go binaries.

1. Basics & Output

PHX features a modern PHP 8+ style syntax compiled directly into native Go machine instructions. Every file must begin with the <?php tag. Comments are supported via standard double-slashes // for single lines.

Variable Declaration & Rules

Variables in PHX are dynamically typed, prefixed with the $ character, and initialized upon assignment. Variable names are case-sensitive and must start with a letter or underscore.

Basic Data Types

Data Type Example Syntax Meaning & Representation Primary Use Case Scenario
Integer $port = 8085; 64-bit signed whole numbers. Loop iterators, array offsets, status codes.
Float $duration = 1.45; 64-bit double precision float numbers. Precise metric logging, calculation multipliers.
String $name = "Vicky"; UTF-8 character sequences inside double quotes. Supports escape sequences like \n. Display text, payload templates, database keys.
Boolean $isActive = true; Logical values (true or false). Feature flags, request filters, completed flags.
Array $items = [1, "two", false]; Ordered lists of items. Supports associative key-value mappings. Record lists, request parameters, database rows.
Null $data = null; Represents the absolute absence of a value. Default values, resetting variables, checking unset attributes.

Standard Output & Concatenation

Use the echo keyword to print messages directly to the standard output. Strings and variables are concatenated using the dot operator (.).

<?php
$name = "Vicky";
$role = "Systems Engineer";

// Concatenating string literal with variables
echo "User: " . $name . " operates as " . $role . "\n";
?>

2. Control Flow

Control flow enables branching execution paths in your application based on logical evaluations.

Conditionals (If / Else If / Else)

Conditional blocks evaluate expressions to truthy or falsy states. Below is the syntax, comparison operators, and logic operations:

Operator Meaning Syntax Example Strictness Behavior
== Loose Equality $a == $b Compares values after type-coercion (e.g. "1" == 1 is true).
=== Strict Equality $a === $b Compares both value and type (e.g. "1" === 1 is false).
!= Loose Inequality $a != $b True if values are not equal after coercion.
!== Strict Inequality $a !== $b True if values or types differ.
<, >, <=, >= Comparison $a <= $b Relational order comparisons.
&&, ||, ! Logical Operators $a && !$b Logical AND, OR, and Negation.

Example: Routing Guard Check

<?php
$auth = "Bearer Token123";
$role = "admin";

if ($auth === "Bearer Token123" && $role === "admin") {
    echo "Access granted to admin panel.\n";
} else if ($auth === "Bearer Token123") {
    echo "Access limited to user dashboard.\n";
} else {
    echo "Access denied: Unauthorized.\n";
}
?>

Switch Match Statements

Evaluate a value sequentially against multiple matching expressions. Supports strict matching and a fallback default option.

<?php
$status = 404;

switch ($status) {
    case 200:
        echo "OK - Request successful.\n";
        break;
    case 404:
        echo "Not Found - Endpoint does not exist.\n";
        break;
    case 500:
        echo "Internal Error - Server failure.\n";
        break;
    default:
        echo "Unknown HTTP status code.\n";
        break;
}
?>

3. Loops

Loops execute statements repeatedly while a condition remains true. Loop control keywords break (exit immediately) and continue (skip to next iteration evaluation) are supported.

While Loops

Evaluates the condition before running the iteration block. Ideal when the total iterations are not known beforehand.

<?php
$ch = channel(5);
send($ch, "Job A");
send($ch, "Job B");
send($ch, "stop");

// Process items until "stop" token is received
while (true) {
    $item = receive($ch);
    if ($item === "stop") {
        break; // Exit the loop instantly
    }
    echo "Processing: " . $item . "\n";
}
?>

Do-While Loops

Executes the iteration block at least once before checking the condition. Useful for retry attempts.

<?php
$retryCount = 0;
do {
    echo "Attempting to sync database pool...\n";
    $retryCount++;
} while ($retryCount < 3);
?>

For Loops

Standard counter iteration loops comprising an initializer, a condition check, and a step operation. Ideal for processing array data.

<?php
$todos = [
    ["id" => 1, "title" => "Deploy server"],
    ["id" => 2, "title" => "Run tests"]
];

for ($i = 0; $i < count($todos); $i++) {
    $t = $todos[$i];
    echo "Todo " . $t["id"] . ": " . $t["title"] . "\n";
}
?>

4. Functions & Closures

Functions isolate and modularize code. Arguments are passed by value and can return dynamic types.

Defining Custom Functions

<?php
function formatResult($code, $message) {
    return "[" . $code . "] Response: " . $message;
}

echo formatResult(200, "Success") . "\n";
?>

Closures & Environment Capture (use keyword)

Closures are anonymous, callable function variables. To access variables from the outer scope, you must capture them explicitly using the use keyword, which copies the variable value into the closure's scope.

<?php
$prefix = "LOG_EVENT";
$logger = function($msg) use ($prefix) {
    echo $prefix . ": " . $msg . "\n";
};

$logger("User logged in successfully"); // Output: LOG_EVENT: User logged in successfully
?>

5. Classes & Object-Oriented Programming (OOP)

PHX translates classes and inheritance directly to Go objects and closures, delivering object-oriented capabilities at compiler level.

Syntax Definition

Classes declare fields (prefixed with access levels like public) and methods. Use $this-> to reference properties or invoke methods belonging to the instance.

Constructor & Instantiation

The constructor method must be named __construct. Instantiate classes using the new keyword.

<?php
class Service {
    public $name;
    public $active;

    public function __construct($name) {
        $this->name = $name;
        $this->active = true;
    }

    public function status() {
        if ($this->active) {
            return $this->name . " is active";
        }
        return $this->name . " is offline";
    }
}

class Microservice extends Service {
    public $port;

    public function __construct($name, $port) {
        $this->name = $name;
        $this->port = $port;
        $this->active = true;
    }

    public function status() {
        return $this->name . " is running on port " . $this->port;
    }
}

$api = new Microservice("AuthService", 8081);
echo $api->status() . "\n"; // Output: AuthService is running on port 8081
?>

6. Exceptions

PHX maps try-catch blocks and throw statements directly to Go's panic-recover runtime. If an exception remains uncaught, it outputs a clean, PHP-style fatal error message to Stderr instead of a raw Go panic trace.

Syntax & Execution

<?php
try {
    echo "Running critical logic...\n";
    throw new Exception("MySQL Database Connection Failed");
} catch (Exception $e) {
    echo "Handled error gracefully: " . $e->getMessage() . "\n";
}
?>

7. File Operations

File inputs and outputs are managed through the built-in File class, bridging PHP operations to Go's os and bufio utilities.

Class: File Methods Reference

Method Arguments Returns Description / Actions
open(string $path, string $mode) $path (file path), $mode ("r": read, "w": write, "a": append) bool Opens the file. Throws exception if path or permissions are invalid.
write(string $content) $content (string payload) bool Writes the contents to the file. Requires file opened in write/append modes.
readLines() None array Reads the entire file contents and returns an array of strings representing each line.
close() None bool Closes the file descriptor, flushing changes to the physical disk.

Example: App Log Writer

<?php
$f = new File();

// Writing log event
$f->open("app.log", "a");
$f->write("LOG EVENT: Auth verified at 12:00PM\n");
$f->close();

// Reading log history
$f->open("app.log", "r");
$lines = $f->readLines();
$f->close();

echo "Total Logs Recorded: " . count($lines) . "\n";
?>

8. Concurrency & Channels

PHX features built-in Go concurrency abstractions. You can execute tasks asynchronously, sync execution paths, and pass messages safely using channels.

Syntax & Definitions

Spawn: The spawn(closure) statement runs the closure concurrently in a Go goroutine and returns a Thread object. The compiler ensures safe type isolation.

Join: Call $thread->join() to block the parent thread until the spawned task completes, returning its result.

Channel: Created via channel(capacity). Pass messages using send(chan, val) and read using receive(chan).

Example: Parallel Scraper & Synchronization

<?php
// Spawn a background worker task
$worker = spawn(function() {
    sleep(1);
    return "Fetched remote API payload";
});

// Perform local operations concurrently
echo "Computing database results locally...\n";

// Await worker response
$data = $worker->join();
echo "Task Result: " . $data . "\n";
?>

Example: Inter-Thread Messaging via Channels

<?php
$queue = channel(10); // Channel of capacity 10

// Worker thread
spawn(function() use ($queue) {
    for ($i = 1; $i <= 3; $i++) {
        send($queue, "Data Packet #" . $i);
        sleep(1);
    }
    send($queue, "EOF");
});

// Listener thread
while (true) {
    $msg = receive($queue);
    if ($msg === "EOF") {
        break;
    }
    echo "Listener Received: " . $msg . "\n";
}
?>

9. High-Performance HTTP, Database Connection Pools & NoSQL

PHX includes an enterprise-grade, high-performance event-based HTTP server module. Built on top of Go's native asynchronous network poller, it automatically utilizes goroutines to multiplex and handle millions of concurrent HTTP requests efficiently. It supports an advanced middleware pipeline, regex-based routing with dynamic parameters, persistent database connection pooling, and a memory-backed cache system.

Class: HttpServer Methods Reference

Method Arguments Returns Description / Actions
configure(array $config) $config: associative array (e.g. ["workers" => 8, "max_connections" => 10000]) HttpServer Configures server concurrency thresholds and worker pool configurations. Supports chaining.
use(function|string $middleware) $middleware: callback closure function($req, $res, $next) or Middleware class name HttpServer Registers a middleware handler into the request lifecycle pipeline. Supports chaining.
router() None Router Returns the primary routing instance.
database(array $pools) $pools: config array of database details (host, user, database, driver) HttpServer Initializes named database pools. Supports MySQL and PostgreSQL drivers. Supports chaining.
db(string $poolName) $poolName: name of database pool (default is "default") Database Resolves and returns a Database connection wrapper from the specified pool.
cache() None Cache Returns the global thread-safe in-memory cache singleton.
listen(int $port) $port: port number null Blocks Starts the HTTP server. Binds to the TCP port and handles incoming connections.

Class: Router Methods Reference

Method Arguments Returns Description / Actions
get|post|put|delete(string $path, function $handler) $path (route pattern matching), $handler (closure callback) Router Registers a route with dynamic parameters parsed within curly braces {name}. Supports chaining.
group(string $prefix, function $callback) $prefix (url group prefix), $callback (closure group setup) Router Creates a route group matching prefixes (e.g. /api/v1). All sub-routes inherit this prefix. Supports chaining.

Class: HttpRequest Methods Reference

Method Returns Description / Actions
getMethod() string Returns the request method (e.g. "GET", "POST").
getPath() string Returns the path (e.g. "/users").
getUri() string Returns the raw URI query string.
getHeaders() array Returns request headers as an associative array.
getQuery() array Returns parsed query string variables.
getBody() string Returns the raw string request payload body (e.g. raw JSON).
getParam(string $name) string Retrieves value of a matched path parameter from the route pattern.

Class: HttpResponse Methods Reference

Method Arguments Returns Description / Actions
status(int $code) $code: HTTP status code HttpResponse Sets response status code. Supports chaining.
header(string $name, string $value) $name: header key, $value: value HttpResponse Sets response HTTP headers. Supports chaining.
write(string $content) $content: string body block HttpResponse Writes a chunk of content to output stream. Supports chaining.
end(string $content = "") $content (optional final string) null Sends content, flushes headers, and terminates connection stream.
json(array|val $value) $value: array or printable value null Serializes the value into JSON, sets content-type to application/json, and calls end().

Class: Cache Methods Reference

Method Arguments Returns Description / Actions
get(string $key) $key: cache key val|null Retrieves value. Returns null if key has expired or does not exist.
set(string $key, val $value, int $ttlSeconds) $key, $value, $ttlSeconds (default is 0: persistent) bool Stores value with an expiration timer.
delete(string $key) $key: cache key bool Removes key from the in-memory cache map.

Built-in Relational Database Clients (MySQL & PostgreSQL)

MySQL and PostgreSQL objects execute database commands. You can connect, run queries that return record arrays, or execute write transactions.

Method Arguments Returns Description / Actions
connect(string $host, string $user, string $pass, string $db, int $port) $host, $user, $pass, $db, $port (port argument is optional for Postgres: defaults to 5432) bool Connects to the database server. Automatically creates the database if it does not exist.
query(string $sql) $sql: SQL select statement array Runs query and returns an associative array representing all rows.
exec(string $sql) $sql: SQL statement (insert/update/delete) bool Executes command statement. Returns true if executed successfully.
close() None bool Closes active connection pool handle.

Example: Standalone MySQL Operations

<?php
$db = new MySQL();
$db->connect("localhost:3306", "root", "password", "test_db");

// Write schema
$db->exec("CREATE TABLE IF NOT EXISTS users (id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(255))");
$db->exec("INSERT INTO users (name) VALUES ('Vicky')");

// Read rows
$rows = $db->query("SELECT * FROM users");
for ($i = 0; $i < count($rows); $i++) {
    $row = $rows[$i];
    echo "User ID: " . $row["id"] . ", Name: " . $row["name"] . "\n";
}
$db->close();
?>

Built-in Document Database Client (MongoDB Mock)

MongoDB object offers an easy-to-use API to persist and find document arrays. It uses an internal high-performance structured memory engine.

Method Arguments Returns Description / Actions
connect(string $uri) $uri (connection string) bool Connects to the database instance.
selectDatabase(string $dbName) $dbName: database name bool Selects current working database context.
selectCollection(string $collName) $collName: collection name bool Selects current collection context.
insertOne(array $document) $document: associative array bool Inserts a document into the current collection context.
find(array $filter) $filter: key-value filter parameters (optional) array Searches and returns matching documents in collection.
close() None bool Closes current MongoDB session.

Example: Standalone MongoDB Operations

<?php
$mongo = new MongoDB();
$mongo->connect("mongodb://localhost:27017");
$mongo->selectDatabase("production");
$mongo->selectCollection("products");

// Insert document
$mongo->insertOne([
    "sku" => "PROD-100",
    "name" => "PHX Server License",
    "price" => 299
]);

// Find document
$results = $mongo->find(["sku" => "PROD-100"]);
if (count($results) > 0) {
    $product = $results[0];
    echo "Found Product: " . $product["name"] . " costing $" . $product["price"] . "\n";

$mongo->close();
?>

Enterprise Cloud Service Example

Below is a production-ready application using event-based routing, database pooling, caching, and a logging middleware pipeline:

<?php
$server = new HttpServer();

// 1. Configure the server runtime
$server->configure([
    "workers" => 8,
    "max_connections" => 10000,
    "keep_alive" => 60
]);

// 2. Logging middleware
$server->use(function($req, $res, $next) {
    $method = $req->getMethod();
    $path = $req->getPath();
    $start = microtime(true);
    
    $nextRes = $next($req);
    
    $duration = (microtime(true) - $start) * 1000;
    echo "[" . $method . "] " . $path . " - processed in " . $duration . "ms\n";
    return $nextRes;
});

// 3. Database connection pool
$server->database([
    "default" => [
        "driver" => "mysql",
        "host" => "localhost:3306",
        "database" => "phx_db",
        "user" => "root",
        "password" => "password",
        "pool_size" => 15
    ]
]);

// 4. Routes definitions
$router = $server->router();

$router->get("/", function($req, $res) {
    $res->status(200)->end("
PHX Enterprise Cloud Engine Running
");
});

$router->group("/api/v1", function($group) use ($server) {
    
    // Database query route
    $group->get("/users/{id}", function($req, $res, $id) use ($server) {
        $db = $server->db("default");
        $rows = $db->query("SELECT id, name, role FROM users WHERE id = " . $id);
        
        if (count($rows) === 0) {
            $res->status(404)->json(["error" => "User not found"]);
            return null;
        }
        $res->json($rows[0]);
    });

    // Thread-safe Cache route
    $group->get("/stats", function($req, $res) use ($server) {
        $cache = $server->cache();
        $counter = $cache->get("requests_counter");
        
        if ($counter === null) {
            $counter = 1;
        } else {
            $counter = $counter + 1;
        }
        
        $cache->set("requests_counter", $counter, 60); // Cache for 60 seconds
        
        $res->json([
            "total_visits" => $counter,
            "cached" => true
        ]);
    });
});

$server->listen(8085);
?>

10. Compiler CLI Usage

PHX packages a compilation command-line tool. You can execute files on-the-fly or compile them into binary executables with no external runtime dependencies.

Execution Modes

Command Primary Action / Meaning Use Case Scenario
bin/phx run app.php Parses, compiles to Go, builds a temporary binary, and runs it on the fly. Local developer iterations, rapid prototyping.
bin/phx build -o my_app app.php Compiles to Go and compiles a native standalone binary inside the output file path. Docker container distributions, staging/production deployments.

Example: Creating Standalone App

$ bin/phx build -o production_server todo_app/index.php
$ ./production_server
Todo App listening on http://localhost:8085
PHX Server listening on http://localhost:8085
