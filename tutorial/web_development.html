Web Development & MVC Guide

Learn how to build production-grade web applications in PHX using the Model-View-Controller (MVC) architectural pattern.

1. Introduction to MVC

The Model-View-Controller pattern divides an application into three main components:

  • Model: Manages the data, database queries, and business rules (e.g. users, tasks).
  • View: Responsible for the visual presentation and layout of the interface (e.g. HTML pages, JSON formats).
  • Controller: Binds the application together. It processes incoming requests, interacts with the model, and selects a view to render.

By keeping these components isolated, your web application remains scalable, easier to test, and highly maintainable as the codebase grows.

2. Standard Project Structure

For a production PHX application, we recommend organizing your files using the following layout:

my_app/
├── index.php             # Main application bootstrap entry-point
├── config/
│   └── database.php      # DB connection pool setup & schema migrations
├── middleware/
│   ├── CORSMiddleware.php
│   └── AuthMiddleware.php
├── controllers/
│   ├── AuthController.php
│   └── TodoController.php
├── views/
│   ├── layout.php        # Common HTML layout (headers, footers)
│   ├── login.php
│   ├── signup.php
│   └── dashboard.php
└── routes/
    └── web.php           # HTTP request routing definitions

3. Database Connection & Schema Setup

Inside config/database.php, we establish connection pools and define helper routines. We can write auto-migration blocks inside a try-catch statement to automatically create required database tables during server startup.

<?php
// config/database.php

$db_host = "127.0.0.1:3306";
$db_user = "root";
$db_pass = "MyNewPass@123";
$db_name = "todo_app";

// 1. Auto-migration / DB creation routine
try {
    $db = new MySQL();
    echo "Connecting to MySQL for migration...\n";
    $db->connect($db_host, $db_user, $db_pass, $db_name);
    
    // Create users table
    $db->exec("CREATE TABLE IF NOT EXISTS users (
        id INT AUTO_INCREMENT PRIMARY KEY,
        username VARCHAR(255) UNIQUE NOT NULL,
        password VARCHAR(255) NOT NULL
    ) ENGINE=InnoDB");
    
    // Create todos table
    $db->exec("CREATE TABLE IF NOT EXISTS todos (
        id INT AUTO_INCREMENT PRIMARY KEY,
        user_id INT NOT NULL,
        title VARCHAR(255) NOT NULL,
        completed BOOLEAN NOT NULL DEFAULT FALSE,
        FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
    ) ENGINE=InnoDB");
    
    $db->close();
    echo "Database successfully verified.\n";
} catch (Exception $e) {
    echo "Database Migration Error: " . $e->getMessage() . "\n";
}

// 2. Helper to fetch connection factory
function get_db_connection() {
    $conn = new MySQL();
    $conn->connect("127.0.0.1:3306", "root", "MyNewPass@123", "todo_app");
    return $conn;
}

// 3. Helper to escape inputs for raw SQL
function db_escape($str) {
    return str_replace("'", "''", $str);
}
?>

4. Session Validation & Auth Middleware

Authenticating client requests relies on parsing incoming cookies and looking up session identifiers in Cache. Below is how to implement cookie parsing and the AuthMiddleware component:

<?php
// config/database.php (Session Helpers)

function get_session_token($req) {
    $headers = $req->getHeaders();
    $cookieHeader = "";
    if ($headers["Cookie"] !== null) {
        $cookieHeader = $headers["Cookie"];
    } else if ($headers["cookie"] !== null) {
        $cookieHeader = $headers["cookie"];
    }
    
    if ($cookieHeader === "") {
        return "";
    }
    
    $pos = strpos($cookieHeader, "session_token=");
    if ($pos === false) {
        return "";
    }
    
    $tokenPart = substr($cookieHeader, $pos + 14);
    $semi = strpos($tokenPart, ";");
    if ($semi !== false) {
        return substr($tokenPart, 0, $semi);
    }
    return $tokenPart;
}

function get_current_user_id($req) {
    $token = get_session_token($req);
    if ($token === "") {
        return null;
    }
    $cache = new Cache();
    return $cache->get("session:" . $token);
}
?>

Now, we implement the AuthMiddleware check, redirecting unauthenticated users to the /login path:

<?php
// middleware/AuthMiddleware.php

class AuthMiddleware {
    public function handle($req, $res, $next) {
        $path = $req->getPath();
        
        // Exclude authentication paths from block check
        $isAuthPath = false;
        if ($path === "/login") {
            $isAuthPath = true;
        }
        if ($path === "/signup") {
            $isAuthPath = true;
        }
        if ($path === "/logout") {
            $isAuthPath = true;
        }
        
        if ($isAuthPath) {
            return $next($req);
        }
        
        // Validate session
        $userId = get_current_user_id($req);
        if ($userId === null) {
            // Unauthenticated! Redirect to login page
            $res->status(302)->header("Location", "/login")->end();
            return null;
        }
        
        return $next($req);
    }
}
?>

5. Creating Dynamic Views & Layouts

Views receive variables from controllers and output structured HTML templates. To maintain consistency, we first create a base layout manager:

<?php
// views/layout.php

class Layout {
    public function render($title, $content) {
        return "<!DOCTYPE html>
<html lang='en'>
<head>
    <meta charset='UTF-8'>
    <title>" . $title . " - PHX Todo</title>
    <style>
        body { font-family: sans-serif; background: #121212; color: #fff; }
        .container { max-width: 600px; margin: 50px auto; padding: 20px; }
    </style>
</head>
<body>
    <div class='container'>" . $content . "</div>
</body>
</html>";
    }
}
?>

Now, we build a specific template (like the Dashboard) that constructs the view payload and wraps it within the Layout:

<?php
// views/dashboard.php

class DashboardView {
    public function render($todos, $username) {
        $total = count($todos);
        
        $content = "<h1>Welcome back, " . $username . "!</h1>
        <p>You have " . $total . " tasks pending.</p>
        
        <form action='/todos/add' method='POST'>
            <input type='text' name='title' placeholder='Add a new task...' required>
            <button type='submit'>Add</button>
        </form>
        <ul>";
        
        for ($i = 0; $i < $total; $i++) {
            $t = $todos[$i];
            $content = $content . "<li>" . $t["title"] . " <a href='/todos/delete/" . $t["id"] . "'>[Delete]</a></li>";
        }
        
        $content = $content . "</ul>
        <p><a href='/logout'>Logout</a></p>";
        
        $layout = new Layout();
        return $layout->render("Dashboard", $content);
    }
}
?>

6. Controllers & CRUD Operations

Controllers contain methods that query the database, process values, and output responses. Here is an example controller handling Todo CRUD operations:

<?php
// controllers/TodoController.php

class TodoController {
    public function showDashboard($req, $res) {
        $userId = get_current_user_id($req);
        
        // Fetch current user details
        $db = get_db_connection();
        $userRows = $db->query("SELECT username FROM users WHERE id = " . $userId);
        $username = "Developer";
        if (count($userRows) > 0) {
            $username = $userRows[0]["username"];
        }
        
        // Fetch user todos
        $todos = $db->query("SELECT id, title, completed FROM todos WHERE user_id = " . $userId . " ORDER BY id DESC");
        $db->close();
        
        $view = new DashboardView();
        $html = $view->render($todos, $username);
        $res->status(200)->header("Content-Type", "text/html")->end($html);
    }
    
    public function addTodo($req, $res) {
        $userId = get_current_user_id($req);
        $post = $req->getPost();
        
        $title = "";
        if (count($post) > 0) {
            if ($post["title"] !== null) {
                $title = trim($post["title"]);
            }
        }
        
        if ($title !== "") {
            $db = get_db_connection();
            $titleEscaped = db_escape($title);
            $db->exec("INSERT INTO todos (user_id, title, completed) VALUES (" . $userId . ", '" . $titleEscaped . "', false)");
            $db->close();
        }
        
        $res->status(302)->header("Location", "/dashboard")->end();
    }

    public function deleteTodo($req, $res, $id) {
        $userId = get_current_user_id($req);
        
        $db = get_db_connection();
        $db->exec("DELETE FROM todos WHERE id = " . $id . " AND user_id = " . $userId);
        $db->close();
        
        $res->status(302)->header("Location", "/dashboard")->end();
    }
}
?>

7. Routing & Server Bootstrapping

Finally, we define the application endpoints in `routes/web.php` and mount the router and middlewares inside the bootstrap index file.

Route Definitions File

<?php
// routes/web.php

$router = $server->router();

// Root route redirects based on session state
$router->get("/", function($req, $res) {
    $userId = get_current_user_id($req);
    if ($userId !== null) {
        $res->status(302)->header("Location", "/dashboard")->end();
    } else {
        $res->status(302)->header("Location", "/login")->end();
    }
});

// Authentication routes
$auth = new AuthController();
$router->get("/login", [$auth, "showLogin"]);
$router->post("/login", [$auth, "login"]);
$router->get("/signup", [$auth, "showSignup"]);
$router->post("/signup", [$auth, "signup"]);
$router->get("/logout", [$auth, "logout"]);

// Todo routes
$todo = new TodoController();
$router->get("/dashboard", [$todo, "showDashboard"]);
$router->post("/todos/add", [$todo, "addTodo"]);
$router->get("/todos/delete/{id}", [$todo, "deleteTodo"]);
?>

App Bootstrap File

<?php
// index.php

// 1. Instantiate Server
$server = new HttpServer();
$server->configure([
    "workers" => 8,
    "max_connections" => 10000
]);

// 2. Include configurations, views, controllers
include "config/database.php";
include "views/layout.php";
include "views/dashboard.php";
include "views/login.php";
include "views/signup.php";
include "controllers/AuthController.php";
include "controllers/TodoController.php";

// 3. Mount Middlewares
$server->use("CORSMiddleware");
$server->use("AuthMiddleware");

// 4. Load Router Routes
include "routes/web.php";

// 5. Start listening
echo "PHX Server starting on port 8086...\n";
$server->listen(8086);
?>
